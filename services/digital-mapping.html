Service 02

Digital Mapping & Web GIS Platforms

Development and deployment of interactive web-based GIS platforms, geospatial dashboards, and real-time spatial visualisation systems for institutional decision-support environments.

Overview

Spatial Intelligence Delivered Through the Web

Web GIS platforms transform raw spatial data into accessible, interactive decision-support environments. A well-designed geospatial portal enables field operators, planners, managers, and executives to interrogate, visualise, and act on spatial information without requiring desktop GIS expertise.

Airfree Geospatial designs and builds custom web mapping applications and GIS portals — from lightweight Leaflet-based operational viewers through to enterprise ArcGIS Online configurations and custom OpenLayers applications with full backend spatial API integration.

Problem Context

Organisations with rich spatial data holdings frequently lack accessible, purpose-built interfaces through which operational staff and decision-makers can interact with that data. Desktop GIS tools require technical training; printed maps are static and rapidly outdated; spreadsheets cannot represent spatial relationships.

A properly architected web GIS platform eliminates these barriers — delivering live, queryable, and spatially rich information to any authorised user with a browser, at any location, on any device.

Solutions Provided

What We Deliver Under This Service

Custom Web Mapping Applications

Purpose-built web mapping applications using Leaflet.js, OpenLayers, or MapLibre GL. Custom layer control, query tools, spatial measurement, feature editing, and data export functionality tailored to operational workflows.

ArcGIS Online & Portal Configuration

Full ArcGIS Online organisation setup, content management, web map and web scene configuration, Experience Builder and Dashboard development, and ArcGIS Enterprise Portal deployment for on-premises environments.

Geospatial Dashboards

Real-time spatial dashboards for operational monitoring and executive reporting. ArcGIS Dashboards, custom web dashboards integrating Plotly, Chart.js, and live WFS data feeds. KPI visualisation with spatial context.

Spatial API Development

RESTful spatial API development using PostgREST, FastAPI, or GeoServer. OGC API Features (OGC API — Features) endpoint configuration, token-based authentication, and spatial query parameter handling for web application backends.

Tile Server & Map Styling

Vector and raster tile server deployment (Martin, TiTiler, GeoServer WMTS). MapLibre GL / Mapbox Style Specification cartographic styling for high-performance, resolution-independent web map visualisation.

Field Data Collection Integration

Integration of field data collection platforms (Survey123, QField, Mergin Maps) with web GIS back-ends. Real-time field observation synchronisation, form-to-database pipelines, and mobile-responsive inspection map interfaces.

Delivery Workflow

Web GIS Development Process

01
Requirements Workshop

User needs analysis, use case definition, data source inventory, authentication and access control requirements, performance expectations, and browser/device compatibility requirements.

02
Wireframe & Architecture Design

Application wireframes, system architecture diagram, data flow specification, API contract definition, and technology stack selection. Client sign-off prior to development commencement.

03
Development & Spatial Data Integration

Iterative frontend development, spatial API integration, layer configuration, symbology and cartographic styling, and feature functionality implementation in staged environments.

04
Testing & UAT

Functional testing, cross-browser compatibility validation, mobile responsiveness testing, load and performance testing, and structured UAT with client user groups.

05
Deployment & Documentation

Production deployment, SSL configuration, access control activation, administrator and user documentation, and post-deployment support period.
